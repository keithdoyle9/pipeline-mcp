# Security Policy

## Supported Versions

This project is currently pre-1.0. The latest commit on `main` is the supported version for security fixes.

## Reporting a Vulnerability

Do not open a public issue for suspected vulnerabilities.

Use GitHub's private vulnerability reporting flow from the repository Security tab. If you cannot use that flow, contact the maintainer through GitHub before disclosing details publicly.

Please include:

- A clear description of the issue.
- Impact and affected components.
- Reproduction steps or proof of concept.
- Suggested mitigations if you have them.

## Response Expectations

The maintainer will aim to acknowledge reports within 5 business days and provide a status update after triage.
